# 途灵 · 现有功能与能力说明

> 本文档根据当前代码实勘整理（`apps/mobile`、`apps/server`、`packages/shared`），描述**已经实现、可运行**的能力，不含未落地的路线图项。  
> 更新日期：2026-07-23

---

## 1. 产品概览

| 项 | 说明 |
|----|------|
| 产品名 | **途灵**（包名 `com.travel.assistant`，scheme `travelassistant`） |
| 定位 | 练手向全栈 AI 旅游助手：把「想去哪玩」变成可生成、可编辑、可分享的行程 |
| 形态 | pnpm Monorepo：Expo App + Node 后端 + 共享类型包 |
| 原则 | App **只**请求自建后端；Claude / 高德 Web 服务 / 和风密钥只在服务端 |

### 技术栈（现状）

| 层级 | 技术 |
|------|------|
| 客户端 | Expo SDK 54 · React Native 0.81 · Expo Router · TypeScript |
| 后端 | Node.js 24+ · Express 5 · Zod · JWT · `node:sqlite` |
| AI | Anthropic Claude SDK（课程网关 Bearer 或官方 API Key） |
| 地图 / 天气 | 高德（Web 服务 + JS API）· 和风天气 |
| 端侧存储 | AsyncStorage（行程本地优先） |
| 工程 | pnpm workspace · `packages/shared` 共享 DTO |

### 典型用户路径

1. 隐私同意 → 产品介绍 →（可选）首次「想去哪儿玩」引导  
2. **规划行程**填表 / 底部 **＋** 短途 / **AI 聊聊** 对话  
3. 预览页生成 → 保存 → **行程**列表 → 详情编辑 / 评价 / 分享  
4. **社区**浏览或发帖；**我的**管理画像、备份、长辈模式、主题、服务器地址

## 2. 移动端能力

### 2.1 启动与引导

| 能力 | 说明 |
|------|------|
| 隐私同意 | 首次必须同意后才能进入主界面 |
| 产品介绍 | 全屏轮播，结束后进入主流程 |
| 首次规划引导 | 「想去哪儿玩」轻弹窗，可快捷跳进行程预览 |
| 冷启动路由 | 默认进入 Tab，避免 Expo 无匹配路由 |

### 2.2 底部导航

五个槽位：

| Tab | 作用 |
|-----|------|
| 规划行程 | 表单规划入口；顶栏可进「AI 聊聊」 |
| 行程 | 本地行程列表与管理 |
| ＋（中间） | **快速短途**弹窗（不是聊天） |
| 社区 | Feed / 发帖入口 |
| 我的 | 名片、设置、服务器调试 |

聊天页路由为 `/(tabs)/chat`，从规划页顶栏进入，不占中间 Tab。

### 2.3 账号与身份

- 设备匿名登录：本地生成并持久化 `deviceId` → `POST /api/auth/device` 换 JWT  
- 改服务器地址会清除 token，下次自动重登  
- 资料：昵称 + 头像（emoji 或本地图）读写云端 profile  
- **没有**手机号 / 邮箱 / 第三方 OAuth

### 2.4 规划行程

表单支持：

- 目的地（城市快捷选、最近目的地）  
- 天数、预算（经济 / 舒适 / 不限 / 自定义）  
- 偏好标签、出行月份、同行人、节奏  
- 出发城市、补充说明  

提交后写入草稿并进入 `/trip/preview` 生成。离线不可生成；蜂窝网络会提示弱网并二次确认。支持恢复未保存预览草稿。

### 2.5 AI 聊天

| 能力 | 说明 |
|------|------|
| 流式回复 | SSE，打字机效果 |
| Markdown | 助手消息渲染 |
| 快捷问题 | chips 一键提问 |
| 会话持久化 | 本地保存最近对话；可开「新对话」 |
| 拍照提问 | 选图 / 拍照，JPEG/PNG base64 上传 |
| 语音 | Web 可用 SpeechRecognition；真机主要靠系统键盘听写 |
| 赞踩反馈 | 本地记录 + 上报后端 |
| 意图跳转 | 识别「帮我规划…」类意图，预填规划表单 |
| 其它 | 复制、引用、失败重试、字数上限约 2000、顶部连通性黄条 |

### 2.6 快速短途

底部中间「＋」：周边城市 chips、1–2 天，直接进预览生成。

### 2.7 行程生成预览（`/trip/preview`）

- 状态：加载（可取消）/ 成功 / 失败  
- 按天展示、天气条、可缩放地图  
- 生成洞察、质量警告、个性化提示  
- 保存本地（并尽力上云）、重新生成、改需求再生成  
- 分享：纯文本 / 长图  
- 未保存返回拦截；生成完成可发本地通知  
- 相似行程提示（覆盖或另存）

### 2.8 行程列表（`/(tabs)/trips`）

- 搜索、筛选（目的地 / 天数 / 预算 / 同行）、排序  
- 置顶、下拉刷新、云端同步状态、删除可撤销  
- 空状态示例行程  
- 「基于此再规划」克隆参数再生成  
- 进入详情 `/trip/[id]`

### 2.9 行程详情（`/trip/[id]`）

| 能力 | 说明 |
|------|------|
| 日程展示 | 按天 Tab；景点 / 餐饮 / 住宿 / 交通等卡片 |
| POI 信息 | 评分、地址、营业时间、缩略图；来源角标（高德实时 / AI 建议） |
| 地图 | 与列表联动高亮 |
| 通勤 | 点间耗时（后端高德估算，失败可本地 Haversine 回退） |
| 外链 | 高德导航 / 小红书 / 大众点评（App scheme 优先，网页兜底） |
| 编辑 | **删除点位、改时间、拖拽排序**（当前不能手动「加点」，需重新生成） |
| 评价 | 单点 + 整程（赞踩 + 标签 + 短评）；需已上云；离线入队、联网补发 |
| 口碑 | 展示 POI 社区口碑聚合 |
| 朗读 | TTS 读当天安排（长辈模式语速更慢） |
| 分享 | 文本 / 长图；可分享到社区（需已上云） |
| 其它 | 重新上传云端、微调 / 重生成 prompt |

### 2.10 地图

- WebView 嵌入高德 JS API 2.0（标点、折线、InfoWindow）  
- Key：`EXPO_PUBLIC_AMAP_JS_KEY` / `SECURITY`（可打包默认值）  
- **不是**高德原生 SDK；**无**离线地图瓦片  
- 无 Key 或离线时显示占位提示

### 2.11 天气

预览 / 详情通过 `GET /api/weather` 拉约 3 天预报；离线不请求。

### 2.12 社区

| 能力 | 说明 |
|------|------|
| Feed | 全部 / 收藏 / 类型筛选；分页；双列卡片；点赞 |
| 详情 | 正文、行程快照、评论、点赞 / 收藏 |
| 发帖 | 类型：行程 / 旅拍 / 评价；可挂已上云行程；可带封面 |
| 演示数据 | 服务端启动可种入演示帖 |

### 2.13 我的 / 设置

- **旅行名片**：头像、昵称、签名、统计看板、年度小结弹窗  
- **旅行画像**：偏好等本机配置 + 云端评价偏好卡片  
- **工具**：行李清单、证件备忘、社区收藏、帮助反馈；「结伴」为即将支持占位  
- **行程管家**：从云端恢复、JSON 文本备份导出（图片 / PDF 导出占位）  
- **更多设置**：长辈模式、增强触觉、出行提醒、小众推荐 / 云同步 / 个性化相关开关  
- **外观**：浅色 / 深色 / 跟随系统  
- **协议**：隐私政策、用户协议摘要  
- **演示服务器**（长按版本号展开）：修改 API Base、测试 `/health`、恢复默认、USB `127.0.0.1`

### 2.14 子页面

| 路由 | 能力 |
|------|------|
| `/profile/luggage` | 本机行李勾选清单 |
| `/profile/documents` | 本机证件备忘（不上云） |
| `/profile/help` | FAQ、邮件反馈、跳转聊天 |

### 2.15 体验增强

| 能力 | 说明 |
|------|------|
| 长辈模式 | 更大字号、触控区、对比；朗读更慢 |
| 主题 | 浅 / 深 / 跟随系统 |
| 触觉 | `expo-haptics`，可关闭 |
| 本地通知 | 行程生成完成通知；出行前提醒（可开关） |
| 网络感知 | online / offline / weak；离线可读已存行程 |

## 3. 后端能力

### 3.1 HTTP API

| 分组 | 方法与路径 | 能力 |
|------|------------|------|
| 健康 | `GET /health` | 存活检查（无鉴权） |
| 认证 | `POST /api/auth/device` | deviceId → JWT |
| | `GET/PATCH /api/auth/profile` | 昵称 / 头像 |
| 聊天 | `POST /api/chat` | 文本（及可选图片）；`?stream=1` 为 SSE |
| | `POST /api/chat/feedback` | 消息赞踩 |
| 行程生成 | `POST /api/trip/generate` | Agent 结构化行程（限流） |
| 行程上云 | `GET/POST /api/trips`，`GET/PUT /api/trips/:id` | 列表 / 创建 / 详情 / 更新 |
| 评价 | `POST/GET /api/reviews`，`GET /api/reviews/preferences` | 写入、回显、偏好汇总 |
| POI | `GET /api/pois/reputation` | 社区口碑聚合 |
| 天气 | `GET /api/weather?city=` | 约 3 天预报（失败可空） |
| 距离 | `GET /api/distance` | 高德步行 / 驾车类估算 |
| 社区 | `GET/POST /api/community/posts` 等 | Feed、发帖、详情、收藏、点赞、评论 |

### 3.2 横切能力

- 业务接口普遍需要 JWT（`Authorization: Bearer`）  
- 聊天 / 行程生成有分钟级限流  
- CORS 开发期全开  
- 监听 `0.0.0.0`，便于局域网 / 公网访问  
- 启动时检查 Claude 凭证，缺失则退出  

### 3.3 数据存储（SQLite）

主要表：`users`、`trips`、`reviews`、`chat_feedbacks`、`community_posts`、点赞 / 收藏 / 评论相关表。

## 4. AI 与外部集成

| 集成 | 状态 | 说明 |
|------|------|------|
| Claude | ✅ | 闲聊 + 行程 Agent（工具轮可调天气 / 搜地点，再输出结构化 JSON 并 enrich） |
| 用户偏好进 Prompt | ✅（浅层） | 历史评价偏好可注入生成；非完整 RAG |
| 社区口碑进工具结果 | ✅ | 搜地点时可带 likeRatio / topTags |
| 高德 Web 服务 | ✅（需 Key） | POI 搜索、距离；无 Key 则相关能力退化 |
| 高德 JS API | ✅ | 客户端地图 WebView |
| 和风天气 | ✅（需 Key + Host） | 城市查询 + 多日预报 |
| 小红书 / 大众点评 | ✅（仅跳转） | 拼 scheme / 网页链接，不爬取、无开放 API |
| 豆包 / DeepSeek 等分流 | ❌ | 当前代码未接入，行程与聊天均走 Claude |
| Postgres | ❌ | 现网为 SQLite |

## 6. 已知边界（相对「完整旅行产品」）

1. **练手级架构**：设备匿名登录、CORS 宽松、单机 SQLite、演示环境常用 HTTP 公网 IP  
2. **无交易闭环**：不订票、不订酒店、无支付  
3. **行程编辑**：可删点 / 改时 / 排序，**不能**在详情里手动加点（需重新生成）  
4. **真机语音输入弱**：主要依赖系统键盘听写，无独立 ASR  
5. **无离线地图**：离线仅能看已保存文字行程  
6. **结伴协同、PDF 导出、深度 RAG、完整多语言**：有占位或脚手架，未完整落地  
7. **社区为轻量演示**：可发帖互动 + 演示 seed，不是完整社交网络  
8. **外部依赖**：缺 Claude 凭证则后端起不来；缺高德 / 和风则 POI / 天气 / 距离能力下降
