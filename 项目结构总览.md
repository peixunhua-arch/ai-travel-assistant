# 途灵 · 项目结构总览

> 最后更新：2026-07-15
> 这是整个 AI 旅游助手项目的结构地图，一份文件查全貌。

---

## 一、项目定位

**途灵**——AI 旅游助手。用户填需求 → Claude 生成结构化行程 → 真实 POI 回填 → 保存/评价/分享/社区。
面向编程新手练手，讲解风格是白话 + 讲原因 + 标坑。

**技术栈**：
- 前端：Expo（React Native SDK 54 + Expo Router 6）+ React 19 + TypeScript
- 后端：Node 24 + Express 5 + `node:sqlite`（内置，零依赖）+ Claude SDK
- 共享：`@travel/shared`（类型包，无构建步骤，直接消费源码）
- 外部 API：Claude（走课程网关）、高德（POI 搜索 + 距离）、和风天气

### 3.2 数据库（8 张表）

```
users ──┬── trips ──┬── reviews（poi_id NULL=整程总评）
        │           └── community_posts（trip_id 可空，删行程 SET NULL）
        ├── chat_feedbacks
        ├── community_posts（发帖人）
        ├── post_likes（user+post 复合主键）
        ├── post_favorites（user+post 复合主键）
        └── post_comments
```

**关键设计**：
- `PRAGMA foreign_keys = ON`（SQLite 默认关外键，必须手动开）
- 评价去重靠**两个部分唯一索引**：`WHERE poi_id IS NOT NULL` 和 `WHERE poi_id IS NULL`（NULL≠NULL）
- `tags` 存 JSON 字符串（SQLite 无数组类型）；`sentiment` 用 INTEGER (1/-1)
- DB 路径用 `import.meta.url` 推绝对路径（避免 cwd 差异）；`.gitignore` 忽略 `data/`

### 3.4 Agent 行程生成（最核心）

```
Phase A（工具轮，最多 10 轮）
  Claude 调 get_weather / search_place 工具
  → 后端执行工具（和风/高德真实 API）
  → 结果喂回 Claude → 循环直到不再调工具

Phase B（收尾轮，最多 6 次）
  Claude 输出行程 JSON
  → extractJson（正则剥围栏）+ JSON.parse + Zod 校验
  → enrichTrip（按 poiId 从工具缓存回填坐标/评分/地址/图片/链接）
  → 质量检查（超半数点没回填 → 判失败）
```

**Prompt Caching**：断点打 system 末尾（tools+system ≈1500 token > 1024 门槛），个人偏好放 user 消息（不进缓存前缀）。

## 五、7 个贯穿全局的设计模式

### 1. 「不留空占位」
数据不存在就 `return null`，绝不画空框。`TripItemCard`/`WeatherStrip`/`CommunityBadge` 都遵守。避免用户看到空格子以为 App 坏了。

### 2. 受控 + 回调组件
`ReviewButtons`/`Stepper`/`SubTabBar` 自己不存业务状态，数据从父组件来、变化回调上交。逻辑集中、组件可复用。

### 3. 本地优先
`saveTrip` 先本地存（永远成功）再上传（失败不阻断）；离线评价入队、联网补发。保证「保存」永远成功。

### 4. 401 自动重登
所有接口遇 401 → 清 token → 重登 → 重试一次。换后端地址也清 token（JWT 绑定后端实例）。

### 5. 超时分级
health 5s、天气/上传 10s、生成 120s。非核心接口失败静默返回空/null/false，绝不抛错打断主流程。

### 6. 定时器必清
`GenerateProgress`/`TypingIndicator`/`Toast`/`CommuteHint` 都在 `useEffect` return 里 `clearTimeout`，防泄漏和卸载后 setState 警告。

### 7. 触觉统一入口
全项目走 `haptics.ts` 的 `tapLight`/`tapSuccess`/`tapError`，不直接调 SDK，方便统一调强度。

## 六、关键坑点速查

| 坑 | 在哪 | 怎么避 |
|----|------|--------|
| SQLite 外键默认关 | `db.ts` | `PRAGMA foreign_keys = ON` |
| NULL≠NULL 破坏去重 | `reviews` 表 | 两个部分唯一索引（WHERE poi_id IS NULL / IS NOT NULL） |
| 陈旧 token 外键失败 | 换 DB 后 | 换 `JWT_SECRET` 触发 401 自动重登 |
| 真机不能用 localhost | `config.ts` | 用电脑局域网 IP 或 USB adb reverse |
| 高德 JS key ≠ Web 服务 key | `config.ts` vs `.env` | 两把不同的 key，别混 |
| 换后端地址必须清 token | `apiBase.ts` | `setApiBaseUrl` 内部清 token |
| curl 带中文乱码 | 测试时 | 用 Node `fetch`（UTF-8），别用 curl |
| ESM import 要带 `.js` | 后端 | `'./db.js'`（本地），`@travel/shared` 不带 |
| reanimated plugin 放最后 | `babel.config.js` | plugins 数组最后一项 |
| whatwg-fetch 与 Hermes 冲突 | `metro.config.js` | 非 web 平台解析到空 shim |
| iOS canOpenURL 要白名单 | `app.json` | 登记 xhsdiscover/dianping/iosamap |
| Android 要允许明文 http | `app.json` | `usesCleartextTraffic: true` |
| view-shot 抓不到节点 | `TripShareCard` | `collapsable={false}` |
| 偏好不能进 system 缓存 | `claude.ts` | 放 user 消息（缓存前缀之后） |

## 八、开发命令

```bash
# 装依赖（根目录）
pnpm install

# 起后端（热重载）
pnpm dev:server          # = pnpm --filter @travel/server dev

# 起前端（Expo）
pnpm dev:mobile          # = pnpm --filter @travel/mobile start
# 或带清缓存
cd apps/mobile && npx expo start --clear

# 类型检查
cd apps/server && npx tsc --noEmit
cd apps/mobile && npx tsc --noEmit
```

**手机连接**：
- 后端地址在 `apps/mobile/src/config.ts`（`EXPO_PUBLIC_API_BASE_URL` 可覆盖）
- Metro：`exp://<电脑IP>:8081`，手机和电脑连同一 WiFi
- 换 WiFi/IP 变了 → 改 `config.ts` 或设环境变量
